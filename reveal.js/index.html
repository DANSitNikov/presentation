What is canvas?

Canvas is an HTML element which can be used to draw graphics via scripting. This can be used to draw graphs, combine photos, or create simple or not animations.

A little bit about canvas history

Canvas was initially introduced by Apple for use in their own Mac OS X WebKit component in 2004, powering applications like Dashboard widgets and the Safari browser. Later, in 2005 it was adopted in version 1.8 of Gecko browsers, and Opera in 2006, and standardized by the Web Hypertext Application Technology Working Group on new proposed specifications for next generation web technologies.

Disadvantages and advantages

Disadvantages

  • Canvas loads the processor and RAM very much
  • Due to a limitation of the garbage collector, there is no way to clear memory
  • You need to handle events with objects yourself
  • Poor performance at high resolution
  • You have to draw each element separately
  • The ability to create special "beacons" on the pages, the so-called Canvas Fingerprinting, to track users on the network

Advantages

  • Unlike SVG, it is much more convenient to deal with a large number of elements
  • Canvas has hardware acceleration
  • You can manipulate each pixel
  • You can apply image processing filters
  • There are many libraries that can help you to use canvas better => big community

Usage of canvas

A canvas consists of a drawable region defined in HTML code with height and width attributes. JavaScript code may access the area through a full set of drawing functions. Some anticipated uses of canvas include building graphs, animations, games, and image composition.

First steps in canvas

To use this technology you need to create canvas element in your html code

Example: <canvas id="canvas" width="150" height="150"></canvas>

At first sight a canvas looks like the img element, with the only clear difference being that it doesn't have the src and alt attributes. Indeed, the canvas element has only two attributes, width and height. These are both optional and can also be set using DOM properties. When no width and height attributes are specified, the canvas will initially be 300 pixels wide and 150 pixels high. The element can be sized arbitrarily by CSS, but during rendering the image is scaled to fit its layout size: if the CSS sizing doesn't respect the ratio of the initial canvas, it will appear distorted.

Connect with JavaScript

Then you need to create a javascript file, to revive your canvas element

const canvas = document.getElementById('canvas'); const ctx = canvas.getContext('2d');

The first line in the script retrieves the node in the DOM representing the canvas element by calling the document.getElementById() method. Once you have the element node, you can access the drawing context using its getContext() method.

Checking for support

The fallback content is displayed in browsers which do not support canvas. Scripts can also check for support programmatically by testing for the presence of the getContext() method. You can check it this way:

const canvas = document.getElementById('canvas'); if (canvas.getContext) { const ctx = canvas.getContext('2d'); ... } else { // if canvas is unsupported you can write some code right here }

Browser support

The element is supported by the current versions of Mozilla Firefox, Google Chrome, Internet Explorer, Safari, Opera and Microsoft Edge.
